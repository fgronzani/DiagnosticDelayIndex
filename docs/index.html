Diagnostic Delay Detector

Detecting Late Presentation Signals in Healthcare Systems

View on GitHub

Introduction

Healthcare systems worldwide face a critical challenge: patients arriving at hospitals with advanced, often preventable diseases. When diagnoses are delayed, patients present with higher clinical severity, require more intensive medical treatment, and face worse outcomes.

How can diagnostic delay be detected when direct patient timeline data is unavailable?

This project uses a proxy-based approach. Instead of tracking individual patient journeys, it analyzes aggregate severity indicators at hospital admission to detect population-level signals of late presentation.

Methodology

Each hospital admission is assigned a composite severity score calculated as follows:

severity_score = (2 × ICU_admission) + (3 × death) + (1 × normalized_LOS)

Diagnostic Delay Index (DDI)

The Diagnostic Delay Index (DDI) is the core metric of this analysis:

DDI = (number of high-severity cases) / (total cases)

High-severity is categorized as cases falling above the 75th percentile of the severity score distribution. An increasing DDI over time suggests patients may be arriving in progressively worse conditions.

Quick Usage

Execute the pipeline via the command line on CSV files containing admission indicators:

# Run on a custom dataset for a specific condition (e.g., Ischemic Stroke)
python main.py --data path/to/dataset.csv --condition I63 --name "Ischemic Stroke"

Alternatively, launch the interactive Streamlit dashboard for real-time adjustments:

streamlit run app.py
